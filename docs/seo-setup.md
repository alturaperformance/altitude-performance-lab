# SEO Setup — AltitudePerformanceLab.com

## Google Search Console (GSC)

### Property URL
`https://altitudeperformancelab.com`
(Use the **URL-prefix** property type, not domain property, unless you have DNS access.)

### Verification Method

The site is pre-wired for the **HTML meta tag** verification method.

1. Log in to [Google Search Console](https://search.google.com/search-console)
2. Add property → URL prefix → `https://altitudeperformancelab.com`
3. Choose **HTML tag** verification
4. Copy the `content` value from the meta tag Google provides (looks like `abc123xyz...`)
5. Open `src/_data/site.js` and set:
   ```js
   googleSiteVerification: "your-token-here",
   ```
6. Commit and push — Netlify will deploy the tag to all pages automatically
7. Return to GSC and click **Verify**

### Sitemap Submission

After verification, submit the sitemap:
- URL: `https://altitudeperformancelab.com/sitemap.xml`
- Path in GSC: **Sitemaps** → paste URL → **Submit**

The sitemap is auto-generated by Eleventy at build time and currently covers:
- Home, Articles index, Tools, Resources, About pages
- 89 published article URLs (as of April 2026)

### robots.txt Status

`/robots.txt` is already correctly configured:
```
User-agent: *
Allow: /

Sitemap: https://altitudeperformancelab.com/sitemap.xml
```

No changes needed.

## Ranking Data Access for SEO Agent

### Option A — Manual CSV Export (immediate, no setup)

1. In GSC: **Performance** → **Search results** → set date range → Export (CSV)
2. Save to `data/gsc-export-YYYY-MM-DD.csv` in the repo
3. SEO Agent reads the CSV and reports on queries, impressions, clicks, average position

**Pros:** Zero infrastructure. Works today.
**Cons:** Manual step required each reporting cycle. No automation.

### Option B — GSC API via Service Account (automated, recommended)

#### Setup Steps

1. **Create a Google Cloud project** at https://console.cloud.google.com
2. **Enable the Search Console API**: APIs & Services → Enable APIs → search "Search Console API"
3. **Create a Service Account**:
   - IAM & Admin → Service Accounts → Create
   - Name: `altitude-seo-agent`
   - Role: not required at project level
   - Create and download JSON key → save as `data/gsc-credentials.json`
4. **Grant GSC access to the service account**:
   - In GSC: Settings → Users and permissions → Add user
   - Email: the service account email (ends in `@...iam.gserviceaccount.com`)
   - Permission: **Full** (needed to read performance data)
5. **Add credentials to Netlify environment** (do NOT commit the JSON key):
   - Netlify → Site settings → Environment variables
   - Add `GSC_SERVICE_ACCOUNT_KEY` = contents of the JSON key file
6. **SEO Agent API call pattern**:

```python
from google.oauth2 import service_account
from googleapiclient.discovery import build
import json, os

creds_json = json.loads(os.environ["GSC_SERVICE_ACCOUNT_KEY"])
creds = service_account.Credentials.from_service_account_info(
    creds_json,
    scopes=["https://www.googleapis.com/auth/webmasters.readonly"]
)
service = build("searchconsole", "v1", credentials=creds)

response = service.searchanalytics().query(
    siteUrl="https://altitudeperformancelab.com",
    body={
        "startDate": "2026-01-01",
        "endDate": "2026-04-18",
        "dimensions": ["query", "page"],
        "rowLimit": 500
    }
).execute()
```

**Pros:** Fully automated. SEO Agent can pull fresh data each run.
**Cons:** Requires Google Cloud setup and credentials management.

### Recommended Approach

**Start with Option A** to validate the SEO Agent's analysis workflow, then migrate to Option B once the workflow is proven and Google Cloud credentials are in place.
